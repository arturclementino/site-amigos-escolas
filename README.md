# AAEGB | Projeto de Reestruturação Digital

Este repositório contém a infraestrutura tecnológica do novo ecossistema digital da **Associação Amigos das Escolas (AAEGB)**. O objetivo deste projeto não é apenas a publicação de um site, mas a construção de uma plataforma de captação de recursos e engajamento orientada a resultados para nossa atuação em Cacine, Guiné-Bissau.

## 1. Visão Estratégica
Não estamos construindo uma vitrine institucional passiva. Estamos desenvolvendo uma **máquina de conversão de parceiros**. O site deve comunicar, acima de tudo, **autonomia, transparência e impacto**. A AAEGB não apenas "leva ajuda"; ela co-constrói capacidade, infraestrutura e futuro junto ao povo Nalu.

## 2. Pilares de Comunicação
Toda alteração de interface ou conteúdo deve ser filtrada por estes princípios:
* **Dignidade sobre Carência:** Nossa narrativa afasta o vitimismo. Mostramos potenciais, talentos e a força da comunidade que estamos co-construindo.
* **Transparência Radical:** Dados, métricas e relatórios devem ser acessíveis. O doador deve saber exatamente como o recurso se transforma em impacto.
* **Arquitetura de Conversão:** Cada seção foi desenhada para conduzir o visitante por uma jornada de confiança: *Quem somos → Por que somos confiáveis → Qual o impacto → Como participar.*

## 3. Dados Fundamentais (Fontes da Verdade)
* **Raízes:** Atuação ininterrupta desde 2009 (15+ anos).
* **Território:** Setor de Cacine, Região de Tombali, Guiné-Bissau.
* **Impacto Atual:** 842+ alunos matriculados, 91 tabancas alcançadas, 100+ famílias apoiadas.
* **Projetos Ativos:** Escola Betel (Base), Casa do Estudante, Rádio Voz de Paz (107.0 MHz), Inclusão Digital, Perfuração de Poços e a campanha prioritária atual: **Ginásio Poliesportivo**.

## 4. Stack Tecnológica
* **Framework:** [Astro](https://astro.build/) (Focado em performance, SEO e escalabilidade).
* **Estilização:** Tailwind CSS (Arquitetura orientada a componentes modulares).
* **Versionamento:** Git/GitHub (Workflow baseado em Pull Requests para revisão técnica e controle de qualidade).

## 5. Diretrizes para Contribuição
* **Branching:** Mantenha a branch `main` protegida. Toda nova funcionalidade, refatoração ou alteração de conteúdo deve ocorrer em *feature branches* e ser submetida via Pull Request.
* **Fidelidade Visual:** Respeite a paleta cromática (tons de verde esmeralda, laranjas de conversão e off-whites editoriais) para garantir consistência em todos os módulos.
* **Performance:** Como visamos parceiros internacionais, a otimização de imagens (diretório `/public/images`) e a limpeza de código são cruciais para a experiência do usuário.
